Bridge Load Segmentation

STAAD.Pro Load-Distance Automation Tool

Quick Start — For Practicing Structural Engineers

This tool automates the tedious manual calculation of partial load overlap across bridge deck strips / STAAD members. Instead of hand-calculating d1/d2 distances for every member-load pair, you define members and loads once — the engine computes all overlaps, generates STAAD.Pro MEMBER LOAD UNI / LIN commands, and exports to Excel.

What Problem Does This Solve?

❌ Without This Tool (Manual Method)

  • Engineer divides bridge width into 10-20 strips in STAAD
  • For each IRC vehicle position, manually calculates which strips are loaded
  • For each loaded strip, calculates d1 (distance from member start to load start) and d2 (distance from member start to load end)
  • For 12 members × 4 load positions = 48 manual calculations
  • High risk of transcription errors → unsafe design or over-design

✅ With This Tool (Automated)

  • Define members once → auto-generate equal strips
  • Click IRC template → loads auto-populated per IRC 6:2017
  • Run Readiness panel shows missing fields instantly before calculation
  • Click Calculate (or keep auto-update ON) → all d1/d2 computed instantly
  • Copy STAAD commands → paste directly into STAAD.Pro editor
  • Excel export for documentation / checking / client submission

Configure the basic parameters of your structure before defining geometry and loads.

Field What It Controls Typical Values

📐 Tab 2: Geometry / Members

Define how the bridge or culvert width is divided into STAAD member strips. Each strip = one STAAD beam/plate element in the transverse direction.

💡 Engineering Context: In STAAD grillage analysis, bridge decks are modeled as a grid of longitudinal and transverse beams. The transverse beams represent strips across the width. This tool computes which portion of each transverse strip receives load.
Column Meaning Notes
Member ID The STAAD member number. Must match your STAAD model exactly. e.g., 1001, 1002, ... Match your STAAD file!
Start (m) Position of the member's left boundary, measured from reference axis (usually left edge = 0). First member typically starts at 0.0
End (m) Position of the member's right boundary. End - Start = member width. Last member ends at Total Width
Group For box culverts: which structural element this strip belongs to. Top Slab, Bottom Slab, Left Wall, Right Wall, or Middle Wall (1/2/3). "General" for bridge decks
📥📤 Import / Export Use Case: Use Members CSV import when geometry is prepared in an external sheet, and export for checker review/record copy. During editing, Width updates live from Start/End so you can catch geometry mistakes early.
⚠️ Critical Check: Member boundaries must be contiguous and non-overlapping. Verify that Member[i].End = Member[i+1].Start for all adjacent pairs. Gaps or overlaps will produce incorrect d1/d2 values.

⬇️ Tab 3: Loads

Define all transverse loads acting across the bridge width. Each load has a start and end position (measured from the same reference axis as members) and an intensity.

Column Meaning Critical Notes
Load ID Unique identifier for traceability in outputs Use descriptive IDs: AA_T1, 70R_T2, EP_LEFT
Case Load case grouping. Loads in the same case are exported under one LOAD n block in STAAD. LC1 = Live, LC2 = Earth, LC3 = Water ...
Start / End Transverse position of the load's left and right edges, using same reference as members. Must use same units and same datum as members.
Intensity Load intensity in kN/m (for UNI) or kN/m² (per meter span). Negative = downward for GY direction (gravity loads). IRC AA: -97.22 kN/m, 70R: -76.59 kN/m
Int. End For trapezoidal loads only (e.g., earth pressure, water pressure). If blank → uniform load (UNI). If filled → trapezoidal (LIN). Earth pressure: 0 at top, K₀γH at bottom
Direction STAAD global direction. GY = vertical, GX = horizontal (earth/water pressure on walls), GZ = longitudinal (braking). 98% of cases use GY for top slab loads
Notes Template metadata and custom tags. Used by sweep auto-detection for vehicle type and bogie classification. Do not delete template metadata unless you plan to tag manually

💡 d1 and d2 Explained:
Member: |——————————————————————|
         ←——d1——→|■■■ LOAD ■■■|←(member end)—→
         ←———————d2————————→

d1 = distance from member START to where load BEGINS on this member
d2 = distance from member START to where load ENDS on this member
Loaded length = d2 − d1

STAAD syntax: <member_id> UNI GY <intensity> <d1> <d2>
Column What It Means QA Check
d1 (Front) Distance from member start to overlap start Must be ≥ 0
d2 (Back) Distance from member start to overlap end Must be > d1 and ≤ member width
Loaded Length d2 − d1 = actual loaded portion on this member Sum across all members for one load = total load span ∩ bridge width
Format (UNI/LIN) UNI = uniform load; LIN = trapezoidal (linearly varying intensity) LIN used for earth/water pressure on walls
🔴 CRITICAL VERIFICATION: Before using results in production, always verify:
  1. Fresh run check: If stale banner is visible, re-run before exporting.
  2. Sum check: Σ(loaded_length) across all members for one load ≈ load span. If not, some load falls outside the bridge.
  3. Boundary check: d1 and d2 must be non-negative and d2 ≤ member width.
  4. Intensity sign: Gravity loads must show negative values for GY direction.
  5. No missing members: Total affected members should match visual inspection of which strips are under the load.
  6. Snapshot check: Case-wise/load-wise summary should match expected load case setup.
🚚 Longitudinal Sweep (same tab): The sweep auto-reads vehicle intent from Loads tab and geometry from Members + Settings, then reports lead position for max sagging, max hogging, and max shear for top slab, bottom slab, side walls, and intermediate walls. Keep increment as needed (0.1m recommended for practical scanning).

📝 Tab 5: STAAD Export

Generates ready-to-paste STAAD.Pro member load commands. Copy the text and paste directly into STAAD.Pro's command file editor (or use "File → Import" in STAAD).

💡 STAAD.Pro Command Format: ; ─── Uniform load (UNI) ───
1003 UNI GY -76.60 0.68 0.70
; Member 1003, uniform, direction GY, intensity -76.60, from d1=0.68 to d2=0.70

; ─── Trapezoidal load (LIN) ───
2001 LIN GX 0.00 18.00 0.00 3.00
; Member 2001, linear, GX, w1=0 at d1=0, w2=18 at d2=3.0
; (Earth pressure: 0 at top, 18 kN/m² at 3m depth)
⚠️ Before Pasting into STAAD:
  • If run readiness is blocked, fix inputs first; export is intentionally blocked
  • Verify the member IDs match your STAAD model exactly
  • Confirm the load directions are correct (GY for slabs, GX for walls)
  • Check that your STAAD model uses the same units as this tool
  • Both d1 and d2 are measured from the member start node, not from the member center

Engineering Formulas & Standards Used

Core Overlap Algorithm

overlap_start = max(member.start, load.start)
overlap_end   = min(member.end, load.end)

if overlap_start < overlap_end  → overlap exists
  d1 = overlap_start − member.start    (front distance for STAAD)
  d2 = overlap_end   − member.start    (back distance for STAAD)
  loaded_length = d2 − d1

Live Load Dispersion Through Fill (IRC 112, Clause 9.3)

b_eff = b_contact + 2 × H_fill × tan(45°)
b_eff = b_contact + 2 × H_fill    (for 45° angle)

Dispersed intensity = Original intensity × (b_contact / b_eff)
Total force is conserved: F = intensity × width = constant

Lateral Earth Pressure (IRC 112 / IS 456)

P(h) = K₀ × γ_soil × h    (varies linearly with depth)
K₀ = 1 − sin(φ) ≈ 0.5    (at-rest, φ = 30° typical)

At top (h=0): P = 0 kN/m²
At bottom (h=H): P = K₀ × γ × H = 0.5 × 18 × 3 = 27 kN/m²

STAAD: member LIN GX 0.00 27.00 0.00 3.00

Important Disclaimers for Professional Use

  1. Independent Verification Required: This tool computes transverse load distribution only. All outputs must be independently verified by a qualified structural engineer before use in design.
  2. Not a Design Tool: This tool does NOT perform structural design (moment, shear, reinforcement). It is a load-distance calculator that feeds into STAAD.Pro for analysis.
  3. IRC Values Are Templates: The IRC load templates are pre-populated per IRC 6:2017. The engineer must verify that the correct loading class, impact factor, and load combinations are applied in their STAAD model.
  4. Impact Factor Not Included: The IRC load intensities shown are without impact factor. The engineer must apply the appropriate impact factor (IF) per IRC 6 Cl. 208 in the STAAD model.
  5. Sign Convention: Negative intensity = downward (GY). Positive intensity on GX = horizontal push on walls. Verify this matches your STAAD model's global axis convention.
  6. Member ID Matching: The STAAD member IDs generated by this tool must match your STAAD model. Incorrect IDs will apply loads to wrong members.
